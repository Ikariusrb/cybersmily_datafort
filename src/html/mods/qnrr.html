These rules are to make netrunning a little faster. They shouldn't take too much real time to do. There are three steps to making a run: make a base link, intrusion, and outcome.
The runner first has to form a base link. That is, what LDL they go through and the trace rating each gives them. If they fail on the security roll the Ref can use the fumble rolls for any of the forts or the one in the book.
Next the runner attempts to break into the fort. The runner gets to choose how, either through a wall or the code gate. The runner and Ref determine success just like they would normally would. If the runner is using a Noisy Program it will affect the outcome roll. For every grid that the Noisy Program is heard from, add half that number to the DIFF of the fort (i.e. Hammer can be heard 10 spaces away, so add 5 to the DIFF of the Outcome DIFF).
And finally, the outcome. A runner adds up all Anti-System, Evasion, and Anti-IC program strengths they are using (or are loaded in their deck) and divides it by 10. This is the Program modifier. A runner can only use one of each type of program. Add the Deck speed, Interface, INT, the program modifier, plus a 1D10 to beat the Difficulty of the Fort.

If successful, the runner accesses what they were looking for. If they fail, consult the Margin of Failure chart below. If a runner rolls a 1, consult the fumble chart in Bartmoss Guide to the Net. Protection programs might help a runner against attacks that hurt the him directly (i.e. physical damage, or INT damage). To determine if the program is successful, the runner needs to roll 1d10 under the strength of the Protection program. If it fails then the runner is affected as the result indicates. As you can see this is a very basic system, but will speed up game play if you have a runner in the party.
